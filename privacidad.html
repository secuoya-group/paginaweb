2. Qué hace la app y por qué pide datos

Secuoya n8n-oauth es una aplicación que ayuda a usuarios autorizados a conectarse a un servidor n8n de manera rápida y didáctica cuando se les concede acceso.

Solicitamos ciertos datos (por ejemplo, identidad básica) para verificar quién eres y habilitar el acceso correcto al servidor n8n asignado.

3. Qué datos recopilamos

Dependiendo del método de autenticación y del proveedor, podemos recopilar:

  • Datos de identidad: nombre, correo electrónico, foto de perfil (si el proveedor la entrega).
  • Datos técnicos: IP aproximada, tipo de navegador, logs de eventos (inicio/cierre de sesión, errores).
  • Tokens de autenticación (OAuth): tokens necesarios para mantener tu sesión o autorizar el acceso.

4. Para qué usamos los datos

Usamos los datos únicamente para:

  • Autenticarte y permitir tu acceso al servidor n8n autorizado.
  • Brindar experiencia guiada y soporte técnico (por ejemplo, diagnósticos básicos).
  • Seguridad: prevenir accesos no autorizados y abuso.

No usamos datos del usuario para publicidad dirigida, venta de datos o fines no relacionados con la funcionalidad del producto. (Esto es consistente con el tipo de restricciones que Google exige en verificación cuando se trata de “datos de usuario de Google”). :contentReference[oaicite:4]{index=4}

5. Cómo almacenamos y protegemos los datos

Implementamos medidas razonables para proteger los datos, por ejemplo:

  • Cifrado en tránsito (HTTPS).
  • Controles de acceso y mínimo privilegio.
  • Protección de tokens (por ejemplo, cifrado/secretos gestionados).
  • Monitoreo de logs para seguridad y resolución de incidentes.

6. Con quién compartimos los datos

En general, no vendemos información personal. Podríamos compartir datos únicamente con:

  • El servidor n8n que el usuario está autorizado a usar (para permitir el acceso).
  • Proveedores técnicos necesarios para operar el servicio (hosting, DNS, monitoreo), bajo acuerdos adecuados.
  • Autoridades si la ley lo exige.

7. Retención y eliminación

Conservamos los datos solo el tiempo necesario para operar el servicio y cumplir obligaciones legales o de seguridad. La política debe indicar retención/eliminación y cómo solicitarlo. :contentReference[oaicite:5]{index=5}

  • Tokens OAuth: se conservan mientras la sesión esté activa o hasta que el usuario revoque acceso.
  • Logs: se conservan por un periodo limitado (ej.: 30–180 días) para seguridad y auditoría.
  • Eliminación: puedes solicitar eliminación escribiendo a soporte@secuoya.group.

8. Tus derechos y cómo ejercerlos

Puedes solicitar acceso, corrección o eliminación de tus datos, según aplique. Para ejercer tus derechos, contáctanos en soporte@secuoya.group.

9. Cambios a esta política

Mantendremos esta política actualizada. Si cambiamos cómo usamos los datos, lo indicaremos aquí. Google pide que la política sea vigente y notifique cambios relevantes. :contentReference[oaicite:6]{index=6}
